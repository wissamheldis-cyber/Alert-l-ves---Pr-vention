# Prompt à envoyer à Antigravity

Tu vas travailler sur un starter React/Vite/TypeScript pour le site Alert’Élèves.

## Mission

Transformer cette base en site premium, responsive, institutionnel et clair, en respectant strictement la direction artistique déjà posée.

## Direction artistique obligatoire

- Style Apple-like institutionnel, académique, 2026.
- Fond clair : blanc, gris papier, gris perle.
- Rouge uniquement comme accent fonctionnel, pas décoratif.
- Pas d’image ni de wording centré sur “la puff”.
- Le premier programme doit s’appeler : **Module 01 — Nouveaux usages à risque**.
- L’identité doit évoquer : prévention, éducation, confiance, protection, clarté.
- Utiliser des cartes, dossiers, documents, surfaces architecturales, lignes de cadrage, séparateurs, onglets.
- Éviter les boules rouges décoratives et les nœuds orbitaux.
- Éviter la surcharge : chaque section doit avoir une intention claire.

## Architecture produit

Le site a deux parcours :

### 1. Espace établissement & adultes

Objectif : rassurer les collèges, lycées, équipes éducatives et adultes.

Pages :
- Accueil institutionnel ;
- Interventions ;
- Documents ;
- Actualités & veille ;
- Contact / demande d’intervention.

Contenus clés :
- module de prévention ;
- documents téléchargeables ;
- actualités juridiques et prévention ;
- demande d’intervention ;
- éléments de confiance.

### 2. Espace élèves

Objectif : prolonger la prévention après intervention.

Pages :
- Accueil élève ;
- Ressources ;
- Quiz & aide.

Contenus clés :
- ressources courtes ;
- quiz ;
- thèmes de compréhension ;
- call to action aide ;
- ton plus direct, mais jamais enfantin.

## Travail demandé

1. Améliorer le responsive mobile et tablette.
2. Créer un vrai menu mobile.
3. Raffiner les espacements et les contrastes.
4. Ajouter des micro-animations discrètes.
5. Améliorer les sections hero.
6. Remplacer les images externes par des assets propres si nécessaire.
7. Faire en sorte que chaque page soit visuellement cohérente.
8. Nettoyer le code sans casser l’architecture.
9. Garder le projet sans dépendances payantes.

## Interdictions

- Ne pas transformer le site en landing page sombre anxiogène.
- Ne pas centrer l’identité sur un produit précis.
- Ne pas inventer de statistiques réelles.
- Ne pas utiliser de logos institutionnels officiels sans autorisation.
- Ne pas ajouter de système backend complexe sans nécessité.
